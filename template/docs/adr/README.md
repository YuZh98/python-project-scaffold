# Architectural Decision Records

ADRs document architectural decisions that are hard to reverse.

When to write an ADR:
1. **Architectural** in scope (cross-module impact)
2. **Hard to reverse** once shipped
3. Has **at least one viable alternative** worth recording

Otherwise: prefer commit message body or design doc.
